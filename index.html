Key Technical Samples

Selected code from the three main technical layers — the DAX measures that produced the actual figures above, the SQL that drives the vintage analysis, and the Python ML pipeline.

-- ══════════════════════════════════════════════════
-- DAX MEASURES — produced the actual dashboard figures
-- ══════════════════════════════════════════════════

-- Total AUM → £1,952,503,404
Total AUM =
SUM( fact_loans[outstanding_balance] )

-- Arrears Rate 30+ → 0.04 (4.0%)
Arrears Rate 30+ =
VAR _Arrears = CALCULATE(
    SUM( fact_loans[outstanding_balance] ),
    fact_loans[dpd] >= 30,
    fact_loans[is_charged_off] = 0
)
RETURN DIVIDE( _Arrears, [Total AUM], 0 )

-- Interest Income → £125,830,000
Interest Income =
SUMX( fact_payments, fact_payments[interest_portion] )

-- High Risk Loan Count → 214
High Risk Loan Count =
CALCULATE(
    COUNTROWS( fact_loans ),
    fact_ml_scores[score_band] IN {
        "High Risk", "Very High Risk"
    }
)

-- YoY comparison (time intelligence)
AUM YoY % =
VAR _CY = [Total AUM]
VAR _PY = CALCULATE(
    [Total AUM],
    SAMEPERIODLASTYEAR( dim_date[date] )
)
RETURN DIVIDE( _CY - _PY, _PY, 0 )
-- ══════════════════════════════════════════════════
-- ClickHouse: Vintage Roll-Rate
-- Window function lagInFrame tracks DPD migration
-- ══════════════════════════════════════════════════

WITH monthly_dpd AS (
    SELECT
        loan_id,
        toStartOfMonth(snapshot_date) AS report_month,
        origination_month,
        CASE
            WHEN dpd = 0               THEN 'Current'
            WHEN dpd BETWEEN 1  AND 29  THEN '1-29 DPD'
            WHEN dpd BETWEEN 30 AND 89  THEN '30-89 DPD'
            ELSE                            '90+ DPD'
        END AS dpd_bucket,
        outstanding_balance
    FROM fact_loans
    WHERE snapshot_date >= subtractMonths(today(), 24)
),
lagged AS (
    SELECT
        loan_id, report_month, origination_month,
        dpd_bucket AS current_bucket,
        lagInFrame(dpd_bucket, 1) OVER (
            PARTITION BY loan_id
            ORDER BY report_month
            ROWS BETWEEN 1 PRECEDING AND CURRENT ROW
        ) AS prev_bucket,
        outstanding_balance
    FROM monthly_dpd
)
SELECT
    origination_month, report_month,
    prev_bucket, current_bucket,
    countDistinct(loan_id)            AS loan_count,
    round(sum(outstanding_balance), 2) AS total_exposure
FROM lagged
WHERE prev_bucket IS NOT NULL
GROUP BY ALL
ORDER BY report_month, prev_bucket
SETTINGS max_threads = 8;

# GradientBoostingClassifier → AUC 0.91
# 214 loans flagged High/Very High Risk

from sklearn.pipeline  import Pipeline
from sklearn.ensemble  import GradientBoostingClassifier
from sklearn.metrics   import roc_auc_score
import shap

FEATURES = [
    "credit_score", "debt_to_income",
    "employment_length", "loan_amount",
    "interest_rate", "dpd", "num_prev_defaults"
]

model = Pipeline([
    ("scaler", StandardScaler()),
    ("clf",    GradientBoostingClassifier(
        n_estimators=200, max_depth=4,
        learning_rate=0.05, random_state=42
    ))
])
model.fit(X_train, y_train)

# Achieved AUC = 0.91 on holdout test set
auc = roc_auc_score(y_test, model.predict_proba(X_test)[:, 1])

# SHAP: debt_to_income = #1 feature (affordability)
explainer = shap.TreeExplainer(model["clf"])
shap_vals = explainer.shap_values(X_sample)

# Score all loans → 214 in High/Very High Risk bands
df["score_band"] = pd.cut(

// ══════════════════════════════════════════════════════
// Power Query (M) — fact_loans transformation
// Adds DPD_Bucket column used throughout all 4 pages
// ══════════════════════════════════════════════════════

let
    Source = Odbc.Query(
        "dsn=ClickHouse_LendSight",
        "SELECT * FROM fact_loans"
    ),
    TypedTable = Table.TransformColumnTypes(Source, {
        {"loan_id",             type text},
        {"origination_date",    type date},
        {"outstanding_balance", type number},
        {"dpd",                 Int64.Type}
    }),
    // DPD_Bucket powers the funnel on Page 3
    WithDPDBucket = Table.AddColumn(TypedTable, "DPD_Bucket",
        each if   [dpd] = 0       then "Current"
             else if [dpd] <= 29   then "1-29 DPD"
             else if [dpd] <= 89   then "30-89 DPD"
             else if [dpd] >= 90   then "90+ DPD"
             else "Written-Off", type text
    ),
    WithUtilisation = Table.AddColumn(WithDPDBucket, "Utilisation_Pct",
        each if [loan_amount] = 0 then null
             else [outstanding_balance] / [loan_amount],
        type number
    ),
    WithCohort = Table.AddColumn(WithUtilisation, "Origination_MonthKey",
        each Date.StartOfMonth([origination_date]), type date
    )
in
    WithCohort
05 — Key Findings
What the Data Revealed

The dashboard surfaced meaningful analytical insights — demonstrating how a real analyst would interpret and communicate findings to stakeholders.

🏠
Mortgage Dominates the Book
Mortgages represent 89.2% of total AUM (£1.74bn of £1.95bn). This concentration means mortgage arrears management is the single biggest lever for overall book health — a key executive insight visible immediately on Page 1.
📍
Balanced Regional Distribution
London leads at £209M, followed closely by Sheffield (£204M) and Glasgow (£203M). The near-even spread across 10 UK cities indicates no single-region concentration risk — a reassuring portfolio health signal.
📈
Seasonal Arrears Peak: May–June
The arrears line chart shows a clear seasonal spike around May–June across all products, reaching ~0.06–0.07 for Credit Card and Auto Finance. In a real business this would trigger a proactive Q1 collections campaign.
🤖
ML Model Validates Underwriting
The risk band validation table confirms: Deep Sub-Prime customers have 16% arrears vs. 1% for Prime — a 16× gap. This proves the model is genuinely discriminating, not just reflecting labelling, validating the credit scorecard logic.
💡
Affordability Beats Bureau Score
SHAP analysis shows debt_to_income is the single strongest predictor of default — outranking credit_score. This tells the credit committee to weight affordability assessment more heavily than bureau score alone in underwriting decisions.
🔍
214 Loans Need Immediate Action
The ML scoring page surfaces exactly 214 loans in the High Risk or Very High Risk bands. In a live environment these automatically populate a collections prioritisation queue — directly reducing future arrears through early intervention.
